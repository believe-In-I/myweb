# 部署指南

## 为什么需要单独的 server.js 文件

### Vite 的两种模式

1. **开发模式**
   - 使用 `npm run start` 启动
   - 由 Vite 内置的开发服务器提供服务
   - 主要用于本地开发，提供热更新、源码映射等开发功能
   - 配置在 `vite.config.mjs` 中

2. **生产模式**
   - 使用 `npm run build` 构建，生成静态文件到 `dist` 目录
   - 需要一个生产级服务器来提供静态文件
   - 配置在 `server.js` 中

### 为什么不直接在 vite.config.mjs 中配置

1. **职责分离**
%NODE_HOME%\node_cache
%NODE_HOME%\node_global
   - `vite.config.mjs` 负责构建过程和开发服务器配置
   - `server.js` 负责生产环境的服务器配置
   - 这种分离使配置更清晰，便于维护

2. **功能差异**
   - 开发服务器需要热更新、源码映射等功能
   - 生产服务器需要压缩、缓存、安全等优化
   - Vite 的开发服务器不适合生产环境使用

3. **部署灵活性**
   - 单独的 `server.js` 可以在不同环境中部署
   - 可以根据服务器环境进行定制
   - 支持更多生产环境的配置选项

4. **性能优化**
   - 生产服务器可以针对性能进行专门优化
   - 可以配置更复杂的缓存策略
   - 可以集成 CDN、负载均衡等高级功能

## 部署步骤

1. **构建项目**
   ```bash
   npm run build
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **启动生产服务器**
   ```bash
   npm run server
   ```

4. **部署到服务器**
   - 将 `dist` 目录和 `server.js` 文件上传到服务器
   - 确保服务器安装了 Node.js
   - 在服务器上运行 `npm run server`

## 服务器配置说明

### 压缩设置
- 启用了 gzip/Brotli 压缩
- 压缩级别：6（平衡压缩率和性能）
- 仅压缩超过 1KB 的文件
- 仅压缩文本和应用类型的文件

### 缓存策略
- 静态资源（CSS、JS、图片等）缓存 1 年
- HTML 文件不缓存，确保获取最新版本
- 启用了 ETag 和 Last-Modified 验证

### 路由处理
- 所有请求都返回 `index.html`，支持单页应用路由
- 启用了 CORS，支持跨域请求

## 环境变量

可以通过环境变量配置服务器端口：

```bash
PORT=8080 npm run server
```

默认端口为 3000。
